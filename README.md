# Real-Time Vibration Detection (STM32 + LSM6DSOX)

Real-time vibration/anomaly detection on an STM32 Nucleo-F401RE board using
the onboard LSM6DSOX IMU, a CMSIS-DSP FIR filter, and an RMS-based threshold
to drive an LED indicator.

## Status

Final version as submitted for the academic project
("GRUPPO7_PROGETTOFINALE"). Build artifacts (`Debug/`), redundant launch
configurations, and the CMSIS precompiled library binaries (`Drivers/CMSIS/Lib/`)
have been removed to keep the repo lightweight. See Notes below for build
instructions.

## Hardware

- Board: STM32 Nucleo-F401RE (STM32F401RETx)
- Sensor: LSM6DSOX 6-axis IMU (accelerometer + gyroscope), I2C interface
- Output: onboard LED (GPIOA, pin 5) as vibration indicator
- UART2 (115200 baud) for debug output to host

## Pipeline

1. **Sampling.** Accelerometer data is read via I2C in polling mode at
   104 Hz (`LSM6DSOX_XL_ODR_104Hz`), full scale +/-2g.
2. **Axis combination.** At each sample, the maximum of the three
   acceleration components (x, y, z) is taken as the signal of interest.
3. **FIR filtering.** A 53-tap low-pass FIR filter (CMSIS-DSP
   `arm_fir_f32`, coefficients precomputed offline) is applied to blocks of
   104 samples to remove high-frequency noise.
4. **RMS envelope.** A sliding window (window size 40 samples) computes the
   RMS of the filtered signal.
5. **Threshold + actuation.** When the RMS envelope exceeds a fixed
   threshold, the onboard LED (GPIOA pin 5) is turned on; otherwise it is
   turned off. Filtered output and RMS values are also streamed over UART2
   for offline inspection.

## Repository structure

```
Core/
  Inc/                          headers (lsm6dsox driver, math helpers, HAL config)
  Src/
    main.c                       entry point, peripheral init (I2C1, USART2, GPIO)
    lsm6dsox_read_data_polling.c  sensor read loop, FIR filter, RMS, threshold, LED
    lsm6dsox_reg.c/.h              ST IMU register driver (vendor)
    arm_fir_data.c                 FIR filter coefficients/data
    math_helper.c                  CMSIS-DSP helper functions
    stm32f4xx_*                    HAL/system files generated by STM32CubeMX
Drivers/
  CMSIS/                          ARM CMSIS core + DSP library (vendor)
  STM32F4xx_HAL_Driver/            ST HAL driver (vendor)
Debug/                            STM32CubeIDE build output (binaries, .map, .o)
GRUPPO7_PROGETTOFINALE.ioc          STM32CubeMX configuration (pin mux, clocks, peripherals)
STM32F401RETX_FLASH.ld / _RAM.ld  linker scripts
*.launch                            STM32CubeIDE run/debug configurations
.project / .cproject / .settings/  Eclipse/STM32CubeIDE project metadata
```

## Build / flash

Open the project root in STM32CubeIDE (Import > Existing Projects into
Workspace). The `.ioc` file can be opened to inspect or regenerate the pin
and clock configuration. Build and flash via the standard STM32CubeIDE
toolchain to a Nucleo-F401RE.

## Notes

- `Drivers/CMSIS/Lib/` (precompiled ARM/GCC/IAR CMSIS-DSP libraries) is not
  included to keep the repo under 10MB. STM32CubeIDE will download the
  required CMSIS pack automatically when building, or it can be installed via
  STM32CubeMX's software packs manager.
- The active launch configuration is `GRUPPO7_PROGETTOFINALE Debug.launch`.
- FIR coefficients in `lsm6dsox_read_data_polling.c` were generated offline
  (referenced in comments as `offline_test.m`, a MATLAB script not included
  in this repo).
